## Architecture Overview

```
DesignEngine (Orchestrator)
│
├── AssetCatalog        — Static data: fabrics, patterns, palettes + SVG generators
├── CanvasRenderer      — Canvas API wrapper: layer drawing, blend modes, transforms
└── UI Wiring           — Grid builders, slider bindings, save + redirect
```

All code lives in **one modular ES5-compatible file** (`public/js/canvas-tool.js`) loaded via a `<script defer>` tag in the Blade template. No bundler step needed — it's served directly from `public/`.

## Canvas Rendering Logic

### Layer 1 — Fabric Base
```js
ctx.drawImage(fabricImg, 0, 0, canvasWidth, canvasHeight);
```
The fabric SVG is a crosshatch weave texture generated dynamically in `AssetCatalog.fabricImage()` using a color specific to each fabric type.

### Layer 2 — Pattern (Tiled + Transformed)
```js
ctx.globalCompositeOperation = 'multiply';   // blends with fabric texture
ctx.filter = palette.filter;                 // CSS filter for color palette

ctx.translate(cx, cy);                       // move pivot to center
ctx.rotate(rotation * Math.PI / 180);        // apply rotation
ctx.scale(scale, scale);                     // apply scale (0.5–2.0)
ctx.translate(-cx, -cy);                     // restore

const pat = ctx.createPattern(patternImg, 'repeat');
ctx.fillStyle = pat;
ctx.fillRect(-overshoot, -overshoot, ...);   // fill beyond bounds to avoid gaps
```

**Why `multiply` blend mode?** It multiplies pixel values of the pattern with those of the fabric, so the pattern appears to be woven into the fabric rather than painted on top.

### Color Palette System
Palettes apply a **CSS `ctx.filter` string** before drawing the pattern layer:

| Palette  | Filter |
|----------|--------|
| Original | `none` |
| Warm     | `sepia(0.35) saturate(1.3) hue-rotate(-10deg)` |
| Cool     | `saturate(0.8) hue-rotate(160deg) brightness(1.05)` |

## Replacing Placeholder Assets

When real assets are ready, update `AssetCatalog` in `canvas-tool.js`:

### For Fabrics
Replace the `fabricImage()` method return value with real image URLs:
```js
// Before (generated SVG):
fabricImage(fabric) { return 'data:image/svg+xml;...' }

// After (real asset):
fabricImage(fabric) { return `/images/fabrics/${fabric.id}.jpg`; }
```

### For Patterns
Replace the `patternImage()` method similarly:
```js
// Before (inline SVG):
patternImage(pattern) { return 'data:image/svg+xml;...' }

// After (real tileable PNG/SVG):
patternImage(pattern) { return `/images/patterns/${pattern.id}.png`; }
```

> **Important:** Pattern images must be **tileable** (seamless tiles) at their native resolution (recommended: 200×200px or 400×400px PNG with transparent background) for `ctx.createPattern('repeat')` to work seamlessly.

## State Management & Save Flow

### State Object
```js
state = {
    fabricId:  'organic_cotton',   // string | null
    patternId: 'acanthus',         // string | null
    paletteId: 'original',         // always has a default
    scale:     100,                // percent (50–200)
    rotation:  0,                  // degrees (0–360)
}
```

### Save Flow
1. User clicks "Save & Proceed"
2. `DesignEngine._save()` fires a `fetch POST /save-design-choice` with `{ fabric, pattern }`
3. `SurveyController@saveDesignChoice` stores them in Laravel session as `fabric_chosen` / `pattern_chosen`
4. Also stored in `sessionStorage` as `nw_design` (JSON) as a client-side backup
5. `window.location.href = '/survey'` navigates to the survey

### Button Enable Condition
```js
saveBtn.disabled = !(state.fabricId && state.patternId);
```

## Data Flow Diagram

```
User selects Fabric
    → DesignEngine._selectFabric()
    → AssetCatalog.fabricImage()      → SVG data-URI
    → CanvasRenderer.setFabric()      → loads HTMLImageElement
    → CanvasRenderer.render()         → draws Layer 1
    → _updateButton()                 → checks if both selected

User selects Pattern
    → DesignEngine._selectPattern()
    → AssetCatalog.patternImage()     → SVG data-URI
    → CanvasRenderer.setPattern()     → loads HTMLImageElement
    → CanvasRenderer.render()         → draws Layer 1 + Layer 2 (multiply)
    → _updateButton()                 → enables save button

User moves Scale/Rotation sliders
    → DesignEngine (input event)
    → CanvasRenderer.render()         → redraws with new transform

User selects Palette
    → DesignEngine._selectPalette()
    → CanvasRenderer.render()         → redraws with new ctx.filter

User clicks Save
    → DesignEngine._save()
    → fetch POST /save-design-choice
    → Laravel session stores fabric_chosen / pattern_chosen
    → window.location.href = '/survey'
```
